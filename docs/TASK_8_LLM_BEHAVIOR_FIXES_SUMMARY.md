## 日期

## 📋 任务概述

修复 Agent 在使用工具时的两个问题：
1. Agent 拒绝使用网络工具，说"我无法访问网络"
2. Agent 输出文本格式的工具调用，而不是正确地调用工具

---

## 🔍 问题 1：Agent 拒绝使用网络工具

### 症状
```
用户："搜索今天的热搜"
Agent："抱歉，我无法真正访问网络进行实时搜索🔒😔"
```

### 根本原因
**LLM 的训练偏见**
- LLM 在训练时被告知它无法访问网络
- 这个限制被深深嵌入到模型的行为中
- 即使有 MCP 搜索工具可用，仍然会拒绝

### 解决方案
在系统提示词中明确告诉 LLM 它**可以**使用工具访问网络

**修改文件**：`src-tauri/src/agent/react.rs`

**添加的内容**：
```
## 🌐 重要：你的能力

**你拥有强大的工具调用能力！**

你拥有以下能力：
- ✅ 网络搜索：使用 brave_search、fetch 等 MCP 工具
- ✅ 文件操作：使用 read_file、write_file 等工具
- ✅ 命令执行：使用 run_command 工具
- ✅ 代码执行：使用 run_python 工具
- ✅ API 调用：通过 MCP 工具调用各种 API
```

---

## 🔍 问题 2：输出文本格式的工具调用

### 症状
```
[TOOL_CALL]{tool=> "run_command", args => {--command "python -c \"
import requests
...
\""}}[/TOOL_CALL]
```

### 根本原因
1. **LLM 的训练数据中有类似格式**
   - 某些框架使用文本格式的工具调用
   - LLM 误以为这是正确的方式

2. **工具选择错误**
   - 使用 `run_python` 执行网络请求
   - 而不是使用专门的 `brave_search` 工具

3. **提示词不够明确**
   - 没有说明如何调用工具
   - 没有禁止文本格式

### 解决方案
在系统提示词中明确说明工具调用方式和工具选择优先级

**修改文件**：`src-tauri/src/agent/react.rs`

**添加的内容**：

#### A. 明确工具调用方式
```
**重要提示**：
- 当你需要执行操作时，直接调用相应的工具函数
- 不要输出工具调用的文本格式（如 [TOOL_CALL]{...}）
- 不要编写 Python 代码来执行网络请求，使用 MCP 搜索工具
- 系统会自动处理工具调用并返回结果
```

#### B. 工具使用指南
```
## 工具使用指南

### 网络搜索
- ✅ 使用 `brave_search` 或 `fetch` 等 MCP 工具
- ❌ 不要使用 `run_python` 执行 `requests.get()`

### 文件操作
- ✅ 使用 `read_file`、`write_file` 等工具
- ❌ 不要使用 `run_python` 执行 `open()` 和 `write()`

### 命令执行
- ✅ 使用 `run_command` 工具
- ❌ 不要使用 `run_python` 执行 `os.system()`
```

## 🎯 关键要点

### 1. LLM 的训练偏见

**问题**：
- LLM 在训练时被告知它无法访问网络
- 这个限制会影响它的行为

**解决**：
- 在系统提示词中明确告诉它可以使用工具
- 强调和重复这个信息

---

### 2. 工具调用方式

**问题**：
- LLM 可能输出文本格式的工具调用
- 而不是通过 API 调用

**解决**：
- 明确说明如何调用工具
- 禁止输出文本格式

---

### 3. 工具选择

**问题**：
- LLM 倾向于使用通用工具（run_python）
- 而不是专门的工具（brave_search）

**解决**：
- 提供工具使用指南
- 明确优先级（专门的工具 > 通用的工具）
